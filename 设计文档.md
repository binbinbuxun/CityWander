# 城市漫游（City Wander）

## 1. 项目概述与核心定位

- **项目类型**：微信小程序纯前端展示项目（课程期末作业）。
- **核心定位**：一本数字化的独立旅行杂志（参考《Kinfolk》《Cereal》），提供沉浸式城市文化阅读体验。
- **数据来源**：无后端，所有数据由本地 JSON 模拟（配合 `wx.setStorageSync` 实现收藏功能）。
- **收录城市**：东京、上海、曼谷、巴厘岛、巴黎、旧金山、里约热内卢。
- **开发原则**：弱化工具属性，拒绝传统旅游 App 的“攻略/打卡/底栏（Tabbar）”模式，追求极致的留白艺术与摄影作品展示。

## 2. 视觉美学与 Vibe 设定 (Light Mode)

AI 开发者在编写 WXSS 时，必须严格遵循以下视觉规范：

- **色彩系统 (Paper Aesthetic)**：
  - 主背景：暖纸色 `#F9F8F6` 或 `#FAF9F5`。
  - 主文字：深碳灰 `#2A2A2A`（禁忌使用纯黑 `#000000`）。
  - 辅助元素/次要文字：浅灰 `#8E8E93` 或 `#B0B0B0`。
  - 强调色：仅由摄影图片的色彩提供，UI 本身保持中性。
- **字体排印 (Typography)**：
  - 大标题：英文使用优雅的衬线体或粗体无衬线，中文使用思源宋体等衬线体（字号极大 `56rpx`-`72rpx`）。
  - 正文阅读：字号 `28rpx`，行高必须拉大至 `line-height: 1.8` 到 `2.0`，字间距微调 `letter-spacing: 2rpx`，营造呼吸感。
- **空间排版 (Whitespace)**：
  - 取消默认的九宫格与对称对称布局。文字与图片错落穿插。
  - 使用非标比例图片（如 `3:4` 竖图、`16:9` 电影画幅），周边保留宽阔的留白 `padding: 40rpx`。

## 3. 信息架构与导航逻辑 (Information Architecture)

**全局禁用微信原生 Tabbar，所有页面在 `app.json` 中配置 `"navigationStyle": "custom"`。**

- **[主干] 首页 (Home)**：纵向滑动切换七座城市封面。
  - -> 左滑或点击左上角 -> 呼出 **全屏目录页 -> 底部包含 **我的书签 **。
  - -> 点击当前城市封面 -> 进入 **城市详情页。
- **[分支] 城市详情页**：作为枢纽，提供向下深潜的三个独立专题入口：
  - -> **城市故事 (Story)**：长文本阅读。
  - -> **城市风味 (Food)**：美食图文特刊。
  - -> **街区漫游 (Scenery)**：摄影散文。

## 4. 核心页面 Blueprint

## 4.1 首页 (Home - Cover Flow)

- **视觉**：一张带有淡阴影 `box-shadow` 的高质量城市摄影大图居中（非全屏，留有纸张白边），错位叠加超大号城市中英文名。底部有极简“下滑”提示。
- **导航栏**：左上角放置 `≡ INDEX` 极简文字按钮，右上角避开微信胶囊。
- **交互**：垂直滑动切换城市，平滑过渡。向右横滑（或点击左上角）呼出全屏目录页。

## 4.2 全屏目录与收藏入口 (Index & Saved Drawer)

入口：在首页（封面流）的左上角，放置一个极其简约的菜单图标。点击该按钮（或在首页向右侧滑），唤起一个全屏的目录页。

- **视觉**：从左侧滑出的全屏浅色遮罩（带有毛玻璃背景或纯纸色）。
- **排版**：
  - \*   **布局形态 (长方形卡片流)**：    *   摒弃纯文字列表，采用纵向排列的 7 张“横向长矩形卡片 (Horizontal Rectangle Cards)”。    *   卡片背景设定为纯白 `#FFFFFF`，带有极弱的弥散阴影 `box-shadow: 0 10rpx 30rpx rgba(0,0,0,0.03)`，使其微微浮于纸质背景之上。    *   卡片之间留有充裕的间距 `margin-bottom: 30rpx`，避免列表感。 *   
  - **卡片内部 UI 设计 (Ticket Vibe)**：    *   采用 `Flexbox` 横向排版。    *   **左侧**：一张比例精致的城市缩略图（建议 16:9 或 1:1，带 `border-radius: 8rpx` 微圆角）。    *   **中间**：强烈的排版对比。大字号英文名（如 **TOKYO**）叠加在稍微靠下的中文字体（如 **东京**）之上。    
  - * **右侧/角落**：点缀该城市的经纬度坐标（如 `35°41'N 139°41'E`），使用浅灰色极小号的无衬线等宽字体，营造硬核与文艺碰撞的编辑感。
    * * **收藏入口收纳**：    *   在卡片列表的最下方，或者浮层屏幕的右下角空白处，放置一个独立的短矩形按钮或悬浮纯文字：“✦ 我的书签 ”。

## 4.3 城市详情页 (City Hub)

- **视觉**：顶部 Hero Image（全屏宽度，高度 60vh），向下滚动时进入浅色纸质背景。
- **内容**：一段简短的城市引言，随后是三个设计感极强的入口模块（如三张错落排列的竖向卡片，分别通往“故事”、“风味”、“漫游”），替代传统的按钮列表。

## 4.4 独立专题页 (Sub-issues: Food & Scenery)

- **城市故事 (Story)**：长文本阅读。
  - **视觉定位**：一篇发表在高级旅行文学志上的深度散文。纯粹的阅读流，极高的文字排印（Typography）要求。
  - **页面背景**：全局使用主背景色（暖纸色 `#F9F8F6`），无任何多余分割线。

- **城市风味 (Food)**：定位为“食物风物志”。
  - 非菜单列表，而是“大图特写 + 40-80 字文艺食评 + 细线分割的标签（如：*清淡 | 炙烤*）”。
- **街区漫游 (Scenery)**：定位为“城市切片”。
  - 按时间或光线分组（如“清晨的巷”、“夜色天际线”）。
  - 采用单栏长卷排版，每张风光图配极简的一句话说明，图片间用极细的实线 `border-bottom: 1px solid #EAEAEA` 分隔，串联起漫游感。
- **交互**：每个内容区块的右上角，隐藏着一个默认描边的“书签”Icon。

## 4.5 书签页 (Saved - My Clippings)

- **视觉**：排版如同一本剪报本。
- **内容**：按“故事 / 美食 / 风景”分类展示用户点击过书签的内容。保留缩略图与摘要。
- **空状态 (Empty State)**：极简排版，页面中央只留一行浅灰色小字：“把打动你的城市片段，留在这里。”
- 用户需要先登录微信后才能收藏

## 6. AI 开发者执行指令 (Strict Prompt Rules)

在编写代码时，AI 必须严格遵守以下技术与视觉准则：

1. **原生微信开发**：仅使用 `WXML`, `WXSS`, `JS`。绝对禁止引入第三方组件库（如 Vant Weapp, ColorUI 等），确保视觉完全自定义。
2. **CSS 布局**：全面使用 `Flexbox` 替代浮动；针对图文错落排版，使用 `CSS Grid` 和 `position: absolute` 相结合，打破常规流式布局。
3. **微动效 (Micro-interactions)**：
   - 页面加载和滚动时，图片与文字块必须加入优雅的淡入上浮动画 (`fade-in-up`)。
   - 点击收藏 Icon 时，需有微小的缩放回弹动效 (`transform: scale`)。
4. **状态管理闭环**：收藏功能不需要真实后端。在 JS 中监听点击事件，将对应的 Item ID 写入微信本地缓存，在“我的收藏”页面中读取缓存并反向查找 JSON 渲染剪报列表。
5. **适配自定义导航栏**：所有的返回按钮 `back` 需手写。样式应当是极简的单线箭头，确保其能够悬浮叠加在页面的顶部图片或留白上。

## 7. 素材指引（高质量免版权图库）

为了让小程序呈现最佳视觉效果，请前往以下无需翻墙、速度快且完全免费可商用的高清素材网站使用城市摄影图片，以供直接使用网络图片 URL 渲染：

1. **Pexels (pexels.com)**：首选。支持中文，国内访问极快，极具人文氛围，适合找城市全景与街头抓拍。
2. **Pixabay (pixabay.com)**：备选。老牌免版权库，建筑与风光类图片质量极高。
3. **FoodiesFeed (foodiesfeed.com)**：美食版块专属。无需注册即可下载极高质量的餐厅食物特写图。
4. **泼辣有图 (polayoutu.com)**：纯国产开源相册，由优秀摄影师分享，充满真实感和故事性。
